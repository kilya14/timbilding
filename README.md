# ТИМБИЛДИНГ — корпоративный сайт и админ-панель

Проект: учебный/дипломный сайт для компании, которая проводит онлайн- и офлайн-тимбилдинги для команд.

Цель проекта — показать полный цикл:
- лендинг для клиентов (HR, руководители),
- каталог программ тимбилдинга,
- форма заявки,
- простая админ-панель для просмотра заявок и управления программами.

## Функционал

### Публичная часть (для клиентов)

- **Главная страница** (`/`):
    - Hero-блок с иллюстрацией, слоганом и кнопкой «Подать заявку».
    - Блок «Основные цели тимбилдинга» с 4 карточками.
    - Блок «Почему выбирают нас?» с преимуществами.
    - История компании/основателя.
    - Тизер программ (квест «Голодные игры»).
    - Форма «Подать заявку».
    - Контакты и футер.

- **Каталог программ** (`/programs`) — список программ тимбилдинга (квесты, выездные форматы и т.п.).
- **Страница конкретной программы** (`/programs/:slug`):
    - название, краткое описание;
    - формат, длительность, от скольки человек, цена;
    - цели проведения (для каких задач подходит);
    - структура мероприятия по шагам;
    - кнопка «Оставить заявку» со скроллом к форме.

- **Форма заявки**:
    - поля: организация, количество участников, желаемая дата, email, телефон, комментарий, выбор программы;
    - отправка на backend (`POST /api/requests`);
    - сообщения об успехе/ошибке.

### Админ-панель (без авторизации, для учебных целей)

- **Заявки** (`/admin/requests`):
    - таблица всех заявок с сайта;
    - фильтр по статусу: «Все / Новые / В работе / Завершены»;
    - возможность сменить статус заявки (`PATCH /api/requests/:id`);
    - отображаются контакты, организация, количество людей, выбранная программа.

- **Программы** (`/admin/programs`):
    - список всех программ, которые видит клиент на сайте;
    - форма создания новой программы (`POST /api/programs`):
        - slug (для URL),
        - название,
        - формат, длительность, от скольки человек, цена (краткий текст),
        - цели программы (список),
        - структура (список шагов);
    - возможность удалить программу (`DELETE /api/programs/:id`).

> Важно: админ-панель доступна без логина/пароля, так как проект учебный.

## Технологии

### Frontend

- React
- React Router
- Redux Toolkit
- Bootstrap 5
- SCSS (файл `src/styles/tokens.scss` с цветовыми токенами и базовыми стилями)

### Backend

- Node.js
- Express
- MongoDB + Mongoose
- CORS, JSON API
